appengine のローカル開発環境 (eclipse) をスマートフォンで見る

最近 appengine の開発はすっかり Macbook Air + eclipse + GitBox に移行しました。

一応Windows機にも eclipse と TortoiseGitを入れてスタンバってますけど全然使わなくなりました。

eclipse さえあればローカルで大体の画面確認とかも出来るんで便利です。

ただスマホの実機で画面の確認するときに、イチイチ本番環境にデプロイしていると時間がかかって仕方ないです。

という訳で、WiFi経由で eclipse のローカル環境(Jetty)でデバッグ出来るようにします。

1) Macbook Air と スマートフォンを同じ無線LANのアクセスポイントに接続

私の場合常時iPhone と MBAを HTC EVO でテザリングしてますから既にクリアです

アドレスの手打ちはすごく面倒なんで、私は Chrome to Phone や Site to Phone をつかってます。

1) Macbook に Chrome をインストール

2) Chrome にGoogle Chrome to Phone 拡張機能と、Site to Phoneをインストール

3) Android機 には Chrome to Phoneをインスコ。

4) iPhone機 には Site to Phone をインスコ

コレでイチイチデプロイしないでもスマートフォン側から画面確認ができます。
